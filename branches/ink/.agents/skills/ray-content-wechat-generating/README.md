# 公众号内容生成 Skill

把一个灵感或关键词，走完一条流水线，产出可以直接发到公众号的终稿。

中间会自动检查 AI 痕迹、旁白句式、转化闭环，确保内容读起来像真人写的。

---

## 怎么触发

对 Agent 说以下任意关键词即可：

| 你说的话 | 会发生什么 |
|---------|-----------|
| "公众号文案" "内容工厂" | 走完整五步流程（采集→选题→大纲→写稿→检查） |
| "直接出稿" "一键出稿" | 跳过前面三步，直接写稿 + 检查 |
| "A/B 标题" "公众号最终稿" | 从已有素材出终稿 |

## 两种模式

```
Full 模式（给了链接或原文时用）

  你给的素材
    → 初始化
    → 灵感捕获
    → 选题大纲
    → 写初稿
    → 自动检查 → 终稿

Quick 模式（只给关键词时用）

  你给的关键词
    → 初始化
    → 写初稿
    → 自动检查 → 终稿
```

## 产出什么

每次跑完，你会拿到：

1. **5 个标题**（流量向 2 + 转化向 2 + 稳健向 1）
2. **2 版开头钩子**（A 痛点直击 / B 结果承诺）
3. **完整正文**（可直接发布，含 ≥3 个可执行动作）
4. **2 版结尾 CTA**（轻转化 / 强转化）
5. **自检报告**（JSON 格式，列出所有检查项的通过/不通过）

## 跑完的文件在哪

每次运行会在 `linshi/wechat-runs/` 下建一个独立目录：

```
linshi/wechat-runs/
└── quick-工作日志框架-20260325-172500/    ← 一次运行 = 一个目录
    ├── state/
    │   ├── config.json         本次运行的参数
    │   └── progress.json       进度（支持断点恢复）
    ├── planning/               五角色讨论规划（如果有）
    ├── step04-draft/
    │   ├── draft.md            初稿
    │   └── meta.json           标题/钩子/CTA/价值单元元数据
    └── output/
        ├── final.md            ✅ 终稿（可直接发布）
        └── check-result.json   自检报告
```

## 怎么定制

### 选文风档位（Style Profile）

每次触发时，初始化阶段（Step 01）会自动推断或询问你用哪个档位：

| Profile | 适用场景 | 特征 |
|---------|---------|------|
| `practical`（默认） | 工具教程、踩坑记录、配置指南 | 短句、第一人称、禁破折号、禁修辞 |
| `narrative` | 架构思路、产品叙事、IP 故事 | 长句、第三人称、允许破折号、贯穿隐喻 |

也可以在 `config/default.json` 里改默认值：

```json
{
  "style_profile": "practical"
}
```

**注意**：Profile 在初始化时确定，写作中途不可更换。

### 改默认 CTA 和读者画像

编辑 `config/default.json`：

```json
{
  "target_reader": "想尝鲜/想落地的实操人群",
  "conversion_goal": "回复\"部署\"领取资料",
  "default_cta": [
    "回复\"部署\"领取《OpenClaw 部署对比 PDF》",
    "回复\"代搭\"查看远程部署与安全优化服务"
  ]
}
```

### 加自己的写作风格参考

把你觉得"写得对味"的文章（markdown 格式）放到：

```
reference/style-samples/sample-01-标题.md
```

放好后告诉 Agent，它会自动分析你的文章风格，提炼出特征写入 `_index.md`，之后每次写稿都会参照。

### 加新的禁用词

直接编辑 `reference/ai-avoidance.md`，在表格里追加即可。

## 自动检查规则

终稿必须全部通过才会定版：

| 检查项 | practical | narrative |
|--------|:---------:|:---------:|
| 结构完整（标题/开头/正文/CTA） | ❌ 不过 | ❌ 不过 |
| AI 禁用词（共享底线） | ❌ 不过 | ❌ 不过 |
| 旁白句式（"本文将"等） | ❌ 不过 | ❌ 不过 |
| 破折号（—） | ❌ 不过 | ⚠️ 仅报告 |
| 可执行动作 ≥ 3 条 | ❌ 不过 | ❌ 不过 |
| A/B 标题 | ❌ 不过 | ❌ 不过 |
| 贯穿隐喻/喻体 | — | ⚠️ 仅建议 |

不通过会自动修正后重检。
