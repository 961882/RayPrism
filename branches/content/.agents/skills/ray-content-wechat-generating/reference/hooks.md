# 钩子模式

公众号文章的开头钩子决定打开率。以下模式经过验证，适用于实操类公众号。

---

## 钩子测试框架

发布前验证：

1. **停留**：看到这句话会不会继续读？
2. **具体**：有没有具体细节？
3. **承诺**：价值点是否清晰？
4. **缺口**：有没有制造信息差？
5. **可信**：够大胆但不离谱？

---

## 痛点直击型

直接戳读者正在经历的困境。

**模式**：`[具体痛点] + [共情]`

**示例**：

- "搞了三天 MCP 部署，结果 Agent 一个工具都调不通。折腾到凌晨三点，发现是一个配置项写错了。"
- "公众号写了两个月，篇篇都是 200 阅读。选题没问题，写法也认真，就是没人看。"
- "老板让你一周搞定 AI Agent 落地方案。你连 API 怎么调都没搞清楚。"

---

## 结果承诺型

先亮结果，再说怎么做。

**模式**：`[可验证结果] + [方法暗示]`

**示例**：

- "用这套流程，我把公众号单篇阅读从 200 拉到 3000。花了两周，核心就 3 个动作。"
- "团队部署 MCP 的时间从 2 天缩到 30 分钟。不是换了工具，是换了部署思路。"
- "3 个月时间，从零开始做到公众号 5000 关注。没投广告，靠的是内容结构。"

---

## 反常识型

打破读者的固有认知。

**模式**：`[大众认为 X] + [实际是 Y]`

**示例**：

- "所有人都在卷 Prompt Engineering，但真正让 Agent 好用的，是工具链设计。"
- "写公众号阅读量低，问题不是标题不够好，是你的选题方法错了。"
- "大家觉得 AI 写作省时间。我的经验是：用 AI 写初稿，反而要多花 2 小时改。"

---

## 数据型

用具体数字开场。

**模式**：`[具体数据] + [出人意料的上下文]`

**示例**：

- "分析了 50 篇公众号爆款，90% 都用了同一种开头结构。"
- "试了 8 个 AI 写作工具，只有 2 个的输出不需要大改。"
- "上线 3 天，收到 47 条付费咨询。改的只是结尾 CTA 的一句话。"

---

## 故事型

直接把读者拉进场景。

**模式**：`[场景 + 冲突]`

**示例**：

- "客户打电话来说：'你们的 Agent 调用失败了，整条产线停了。'这是我自己写的第一个 MCP Server 上线后的第三天。"
- "凌晨两点，我盯着后台数据发呆。文章发出去 6 小时，阅读量卡在 89。"

---

## 清单型

告诉读者能得到什么。

**模式**：`[数量] + [具体内容] + [承诺]`

**示例**：

- "5 个立刻能用的公众号标题公式，直接套。"
- "Agent 部署踩过的 7 个坑，附解决方案。"
- "给你 3 套 MCP 配置模板，覆盖 90% 的常见场景。"

---

## 钩子长度

公众号钩子：80-150 字。太短没有信息量，太长读者划走。

## 避坑

- 不用问句开头（"你是否想过…"）— 公众号场景效果差
- 不用虚词堆砌（"在当今 AI 快速发展的时代…"）
- 不承诺做不到的事
