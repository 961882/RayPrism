# Style Profile: narrative（叙事散文体）

> 适用场景：架构思路、产品叙事、理念分享、IP 故事
> Profile ID: `narrative`
> 参考样本：`reference/style-samples/sample-01.md`

---

## 1. 句式规则

- 句长：20-80 字为主
- 节奏模式：长句铺陈 + 短句点睛。用 2-3 句长句建立画面感，再用 1 句短句收束
- 段落上限：≤ 5 句/段，允许长段叙事
- 允许排比和对仗结构

## 2. 标点规则

| 标点 | 状态 | 用法 |
|------|------|------|
| — 破折号 | ✅ 允许 | 用于补充说明、语气停顿、插入解释 |
| … 省略号 | ✅ 允许 | 用于留白、余韵、节奏过渡 |
| ？问句 | ✅ 允许 | 可用于反问和设问，增加对话感 |

## 3. 人称与语气

- **人称**：第三人称"Ray"为主，偶尔切回第一人称做点睛
- **立场**：通过叙事隐含。让故事自己说话，不直接说"我认为"
- **语气**：诗化、内敛、有画面感。像在讲一个安静的故事
- **允许**："悄然""轻轻""如…般""那份…"等诗化修辞词
- **禁止**："大家好""今天给大家分享""本文将为您"

## 4. 修辞规则

- **隐喻**：✅ 贯穿全文。每篇文章选定一个喻体（如"王国/贤臣""织布/经纬""编年史"），全文保持一致
- **隐喻一致性**：同一篇文章内不允许切换喻体，上半篇"织布"下半篇不能变"建城"
- **诗化修辞**：✅ 允许，但要求：
  - 每个修辞必须服务于信息传达，不能纯装饰
  - 修辞密度不超过 30%（70% 的句子仍需传递具体信息）
- **技术词**：隐喻包裹后出。例如 `讨论动作记录.md` 写成"一张总表，如编年史的目录"
- **数字**：仍需具体，但可以用修辞包裹。"11 轮对话如 11 层阶梯"
- **工具名**：可以出现，但嵌入叙事而非直列

## 5. 结构公式

```
[场景铺陈：100-200 字，第三人称叙事，建立画面]
[问题揭示：从场景中自然引出核心矛盾]

[编号段落 1-N：每段遵循]
  → 隐喻引入（一句话建立这一段的画面）
  → 技术展开（用隐喻包裹的方式讲清楚做了什么）
  → 金句收束（一句话收住这一段的核心观点）

[踩坑段：用故事讲反例，不直接列"踩坑"]
[数据段：用具体数字收束全文的可信度]
[余韵收尾：开放式结尾，不做总结性排比]
[CTA：诗化包裹但关键词动作必须清晰可执行]
```

## 6. 自检清单

- [ ] 是否有贯穿全文的统一喻体？
- [ ] 喻体是否全文一致（未中途切换）？
- [ ] 修辞是否服务于信息（非纯装饰）？
- [ ] 第三人称叙事为主？
- [ ] 有具体数字/文件名/配置项（即使经过修辞包裹）？
- [ ] CTA 的回复关键词是否清晰可执行？
- [ ] 零 AI 禁用词（共享底线）？
