# Style Profile: practical（实操干货体）

> 适用场景：工具教程、踩坑记录、配置指南、对比评测
> Profile ID: `practical`

---

## 1. 句式规则

- 句长：5-35 字为主
- 长短交替：刻意穿插短句（5-10 字）和中长句（20-35 字），不允许连续 3 句长度接近
- 段落上限：≤ 3 句/段
- 段间必须空行

## 2. 标点规则

| 标点 | 状态 | 替代 |
|------|------|------|
| — 破折号 | ❌ 禁止 | 逗号、冒号、句号、括号 |
| … 省略号 | ⚠️ 谨慎 | 尽量不用，万不得已时可用 |
| ？问句 | ⚠️ 谨慎 | 不用问句开头，偶尔可在正文中使用 |

## 3. 人称与语气

- **人称**：第一人称"我"为主
- **立场**：必须明确。用"我认为""我的做法是""我选了 X 不选 Y"
- **语气**：直接、自信但不吹。像导师跟学生说话
- **禁止**："大家""各位""本文""读者朋友们"

## 4. 修辞规则

- **隐喻**：极少使用。偶尔一个比喻辅助理解即可
- **诗化修辞**：❌ 禁止。不用"如…般""那份…""悄然""轻轻""仿佛"
- **技术词**：直出。直接写文件名、命令、配置项，不做隐喻包裹
- **数字**：必须具体。不说"大幅提升"，说"提升了 40%"
- **工具名**：必须点名。不说"某工具"，说"Claude / Cursor / MCP"

## 5. 结构公式

```
[钩子：80-150 字，痛点直击或结果承诺]
[核心观点：1 句话结论]
[展开 1：场景 + 问题]
[展开 2：做法 + 步骤]
[展开 3：踩坑 + 修正]
[展开 4：结果 + 数据]
[CTA：明确下一步]
```

## 6. 自检清单

- [ ] 第一句就进主题？
- [ ] 句长有变化（短中长交替）？
- [ ] 每段 ≤ 3 句？
- [ ] 零破折号？
- [ ] 有"我认为"或"我的做法"等立场句？
- [ ] 有具体数字/工具名/文件名？
- [ ] 无诗化修辞？
