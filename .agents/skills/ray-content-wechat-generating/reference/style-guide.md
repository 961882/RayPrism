# 文风与排版指南

公众号文章的语气、结构和排版规则。

---

## 语气

### 直接，不绕弯

第一句就进主题。不用"大家好""今天给大家分享"。

**✓** "MCP 部署最常踩的坑是权限配置。"
**✗** "大家好，今天我来和大家分享一下关于 MCP 部署的一些经验…"

### 自信但不吹

说你做了什么、学到什么。不过度修饰。

**✓** "这套流程帮我把出稿时间从 3 天减到 1 天。"
**✗** "这套颠覆性的流程将彻底改变你的写作效率…"

### 有立场

不做两边都对的总结。明确说自己选哪个、为什么。

**✓** "我选 Claude 不选 GPT，因为长上下文场景下它更稳定。"
**✗** "两个工具各有优劣，大家可以根据自己的需求选择。"

### 口语化但不随意

像导师跟学生说话。用通俗的词，但逻辑清晰。

## 排版

### 段落

- 每段 ≤ 3 句
- 段间空行
- 一段一个重点

### 句长

- 刻意变化：短（5-10 字）、中（15-20 字）、长（25-35 字）交替
- 不要连续 3 句以上长度接近

### 列表

- 超过 3 个要点用列表
- 先用 1-2 句叙述铺垫，再列
- 每条一行，不要写成段落

### 加粗

- 关键结论加粗
- 不要整段加粗
- 不要加粗超过全文 10%

### 标题层级

- H1：文章主标题（1 个）
- H2：大段落标题（3-5 个）
- H3：细分段落（按需）
- 不用 H4 及以下

---

## 具体化规则

### 必须有

- 具体工具名（不说"某工具"，说"Claude / Cursor / MCP"）
- 具体数字（不说"大幅提升"，说"提升了 40%"）
- 具体时间（不说"最近"，说"上周三"）
- 具体人（不说"有人说"，说"我的同事老王"）

### 避免

- 模糊比较（"更好""更快"而没有参照）
- 无主语断言（"据说""有人认为"）
- 意义不明的形容词（"高效""优质""强大"）

---

## 结构公式

### 标准结构

```
[钩子：80-150 字]

[核心观点：1 句话结论]

[展开 1：场景 + 问题]
[展开 2：做法 + 步骤]
[展开 3：踩坑 + 修正]
[展开 4：结果 + 数据]

[CTA：明确下一步]
```

### 每段结构

```
[结论]（一句话亮观点）
[证据]（数据/案例/对比）
[动作]（读者可以做什么）
```

- [ ] 第一句就进主题？
- [ ] 有具体数字/名字/工具？
- [ ] 每段 ≤ 3 句？
- [ ] 句长有变化？
- [ ] 有作者立场？
- [ ] 不说"大家""各位"？
- [ ] 不用被禁句式？
