## 1. 环境准备与约定

### 测试环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | macOS（bash / zsh） |
| Python | 3.x（系统自带即可） |
| Git | 已安装 |
| Homebrew | 已安装（用于 fswatch / SpecStory） |
| Node.js / npx | 已安装（用于 SpecStory Agent Skills） |
| RayPrism 框架 | 已 clone 到本地 |

### 约定符号

| 符号 | 含义 |
|------|------|
| ✅ PASS | 结果符合预期 |
| ❌ FAIL | 结果不符合预期 |
| ⚠️ WARN | 部分通过或存在已知限制 |
| 🔧 | 需要人工判断的主观检查 |

### 全局前置条件

```bash
# 确认 RayPrism 仓库克隆位置
RAYPRISM_HOME=/path/to/RayPrism    # 替换为实际路径
cd "$RAYPRISM_HOME"

# 确认分支模板完整
ls branches/
# 预期输出: content  dev  ops  pro  README.md
```

## T1 — 框架环境初始化（init.sh）

### T1.1 首次执行 init.sh

**前置条件**: 全新 clone 的 RayPrism 仓库，无 `.specstory/`、无 `trigger.sh`

**操作步骤**:

```bash
cd "$RAYPRISM_HOME"
bash setup/init.sh
```

**预期结果**:

| 步骤 | 预期输出关键字 | 判定 |
|------|---------------|------|
| ① .specstory/history/ | `✅ .specstory/history/ 已就绪` | |
| ② SpecStory CLI | `✅ specstory 已安装` 或 `✅ SpecStory CLI 安装完成` | |
| ③ SpecStory Agent Skills | `✅ specstoryai/agent-skills 安装完成` | |
| ④ fswatch | `✅ fswatch 已安装` 或 `✅ fswatch 安装完成` | |
| ④ trigger.sh | `✅ trigger.sh 已复制到项目根目录` | |
| ⑤ .env.example | `✅ .env.example 已生成` | |
| ⑥ .env | `ℹ️ 未找到 .env` 或 `✅ .env 已存在` | |
| ⑦ .gitignore | `.env 已在 .gitignore 中` 或 `已将 .env 添加到 .gitignore` | |
| 总结 | `✅ 初始化完成！` | |

**验证**:

```bash
# 确认目录和文件创建

ls -la trigger.sh
ls -la .env.example
```
